## üß† What happens if you give more elements than the array length?

### üîπ `slice(start, end)`

Returns a **new array**, does **not modify** original.

* ‚úÖ If `end` is **greater than array length**, it's **clipped to array.length**.

**Example:**

```js
const arr = [10, 20, 30];

console.log(arr.slice(1, 10)); // ‚Üí [20, 30]
```

üëâ It simply returns elements till the end of the array. No error.

---

### üîπ `splice(start, deleteCount, ...itemsToInsert)`

**Modifies** original array.

* ‚úÖ If `deleteCount` is **greater than remaining elements**, it just deletes till the end.
* ‚úÖ If you **insert more elements** than deleted, it grows the array.

**Example 1 ‚Äì deleteCount too big:**

```js
const arr = [10, 20, 30];

arr.splice(1, 10); // tries to delete 10 elements starting at index 1
console.log(arr); // ‚Üí [10]
```

No error! Just deletes as many as available.

**Example 2 ‚Äì insert more than deleted:**

```js
const arr = [10, 20, 30];

arr.splice(1, 1, 100, 200, 300);
console.log(arr); // ‚Üí [10, 100, 200, 300, 30]
```

It deleted 1 item (`20`) and inserted 3 items.

---

## ‚úÖ Summary Table

| Case                     | `slice()`                  | `splice()`                 |
| ------------------------ | -------------------------- | -------------------------- |
| End index > array length | No error, returns till end | No error, deletes till end |
| Delete more than exists  | Not applicable             | Deletes till end           |
| Insert more than deleted | Not applicable             | Inserts all, array grows   |

## üß† `.flat()` ‚Äî What is it?

`.flat()` is a **JavaScript array method** that **flattens nested arrays** into a single array ‚Äî up to a given depth.

### ‚úÖ Basic Syntax:

```js
arr.flat(depth)
```

* **`arr`** ‚Üí the array to flatten
* **`depth`** *(optional)* ‚Üí how deep to flatten (default is `1`)

---

## ‚úÖ Simple Example

```js
const arr = [1, 2, [3, 4]];
console.log(arr.flat()); // [1, 2, 3, 4]
```

### üîç What happened?

* One level of nesting (`[3, 4]`) was **flattened**

---

## üìê How the `depth` Parameter Works

### üìå Default: `depth = 1`

```js
[1, [2, [3]]].flat();  // [1, 2, [3]]
```

### üîÅ Deeper flattening:

```js
[1, [2, [3, [4]]]].flat(2);  // [1, 2, 3, [4]]
```

### üåå Full flattening:

Use `Infinity` to flatten *completely*:

```js
[1, [2, [3, [4]]]].flat(Infinity);  // [1, 2, 3, 4]
```

---

## ‚ö†Ô∏è Edge Cases to Watch Out For

### 1. Flattens **only arrays**, not objects

```js
[1, { a: 2 }, [3]].flat();  // [1, { a: 2 }, 3]
```

### 2. Skips empty slots (holes)

```js
const arr = [1, , 3, [4, , 6]];
console.log(arr.flat());  // [1, 3, 4, 6]
```

> ‚ùó `.flat()` removes empty slots (like `.filter(Boolean)` in a way)

---

## üîÅ Comparison: `.flat()` vs `.flatMap()`

### üîπ `.flatMap()` = `.map()` + `.flat(1)`

```js
[1, 2, 3].flatMap(x => [x, x * 2]);
// [1, 2, 2, 4, 3, 6]
```

Same as:

```js
[1, 2, 3].map(x => [x, x * 2]).flat();
// [[1,2], [2,4], [3,6]] ‚Üí flat ‚Üí [1,2,2,4,3,6]
```

> ‚ö†Ô∏è `.flatMap()` only flattens **one level**, cannot accept a depth parameter

---

## üéØ Use Cases

### ‚úÖ Remove one level of nesting:

```js
[[1], [2], [3]].flat(); // [1, 2, 3]
```

### ‚úÖ Deep flatten any level:

```js
const deepArray = [1, [2, [3, [4, [5]]]]];
deepArray.flat(Infinity); // [1, 2, 3, 4, 5]
```

### ‚úÖ Clean up messy API data:

```js

arr.flat(2); // [1, 2, 3, 4]
```

## ‚úÇÔ∏è `slice()` ‚Äì Think: ‚ÄúCut a piece of cake, without breaking the rest!‚Äù

### ‚úÖ Purpose:

`slice()` returns a **shallow copy** of a **portion** of an array.

üî∏ It does **not** modify the original array.
üî∏ It takes two optional arguments: `start` and `end`.

---

### üì¶ Syntax:

```js
array.slice(start, end)
```

* `start`: index to start cutting from (inclusive)
* `end`: index to stop (exclusive, i.e., not included)
* If `end` is omitted, it slices till the end of array

---

### üçï Example:

```js
const fruits = ['Apple', 'Banana', 'Cherry', 'Date', 'Elderberry'];

const sliced = fruits.slice(1, 4);
console.log(sliced); // ['Banana', 'Cherry', 'Date']
```

‚úÖ Starts from index `1` (`Banana`)
‚úÖ Ends **before** index `4` (`Elderberry` not included)

---

### üß™ More Examples:

#### 1Ô∏è‚É£ Start only:

```js
fruits.slice(2); // ['Cherry', 'Date', 'Elderberry']
```

#### 2Ô∏è‚É£ Negative indices:

```js
fruits.slice(-2); // ['Date', 'Elderberry']
```

* `-2` means "2 from the end"

#### 3Ô∏è‚É£ Full copy of array:

```js
const copy = fruits.slice(); // copies the whole array
```

---

### üÜö Compare: `slice()` vs `splice()`

| Feature        | `.slice()` (safe)    | `.splice()` (mutates) |
| -------------- | -------------------- | --------------------- |
| Changes array? | ‚ùå No                 | ‚úÖ Yes                 |
| Returns        | New sliced array     | Removed items         |
